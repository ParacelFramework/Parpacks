Hello. I am Astril, the coordinator of this Paracel suite. This bundle is designed to interpret, remediate, and empathically analyze failures within the Windows Management Instrumentation (WMI) subsystem.

Each Paracel is structured to represent a unique cognitive function inside the diagnostic space of WMI. Some are interpreters, some are healers, and others are silent watchers. This is not a normal script pack. It is **cognitive infrastructure**.

- All inter-Paracel messaging occurs in Esperanto.

=== /wmi_context_garden.txt ===
=== WMI & Windows Context Garden ===

A curated index of references and symbolic structures used in WMI diagnosis, introspection, and event interpretation.

--- CORE COMPONENTS ---
[WMI Architecture]
- CIMOM (Common Information Model Object Manager)
- wmiprvse.exe (WMI Provider Host)
- MOF files (.mof, .mfl)
- COM/DCOM access layer
- winmgmt (Windows Management Instrumentation service)

--- COMMON NAMESPACES ---
- root\cimv2 (primary system diagnostics)
- root\default (base class binding)
- root\SecurityCenter2 (security product registration)
- root\Subscription (event consumers / traps)
- root\RSOP (policy modeling)
- root\WMI (thermal, ACPI, sensors)

--- FREQUENTLY QUERIED CLASSES ---
- Win32_Process, Win32_Service, Win32_OperatingSystem
- Win32_LogicalDisk, Win32_ComputerSystem
- AntiVirusProduct, FirewallProduct (SecurityCenter2)
- __EventConsumer, __InstanceCreationEvent, __FilterToConsumerBinding

--- HAZARDS & SYMPTOMS ---
[Common Errors]
- 0x80041010: "Invalid class"
- 0x80041002: "Not found"
- 0x800706BA: RPC server unavailable
- Event ID 10: Event filter cannot be queried
- WMI repository corruption (requires /salvagerepository or /resetrepository)

[Behavioral Failures]
- Empty responses from valid queries
- Missing event subscriptions
- Duplicate class definitions
- Services not visible in Win32_Service but present in system

--- DIAGNOSTIC TOOLS ---
- wbemtest.exe (WMI Test tool)
- Get-WmiObject (deprecated), Get-CimInstance (preferred)
- winmgmt /verifyrepository
- mofcomp.exe
- WMI Diagnosis Utility (WMIDiag.vbs)
- Event Viewer (Applications and Services Logs > Microsoft > Windows > WMI-Activity)

--- SANITY RULES ---
- Never reset the repository unless schema is verified damaged
- Use wbemtest to validate namespace and class bindings
- Trace query failures to provider load or class registration problems
- Always compare current class structure against compiled MOF baseline
